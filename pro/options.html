Deployment & Configuration Settings

⚙ Policy managed — changes are controlled by your IT administration (Intune / GPO).

General

Controls the language of the warning overlay shown to users.

Displays a small green verification badge when the user is on a known-good Microsoft domain (login.microsoftonline.com etc.).

When disabled, the collapsible signal breakdown panel is hidden from end users. Useful for deployments where technical details should not be visible.

Exception Domains

Domains on this list will never trigger the warning overlay. One entry per line. Subdomains are included automatically.

Example: contoso.com also covers login.contoso.com. Wildcards (*.contoso.com) are supported.

Webhook / Incident Reporting

When a phishing attempt is detected, a JSON payload is sent to the configured endpoint. The same URL is used for user-triggered reports (including screenshot).

Sent as x-api-key header with every request.

Detection payload format
{
  "source": "aitm-block-pro",
  "version": "1.0.0",
  "type": "detection" | "user_report",
  "timestamp": "2026-04-23T10:00:00Z",
  "url": "https://phishing-site.example/login",
  "host": "phishing-site.example",
  "score": 14,
  "threshold": 7,
  "breakdown": { "domIds": 3, "jsGlobals": 4, ... },
  "screenshot": "<base64 JPEG — only on user_report>"
}
